# Deploying changes

The site is hosted on **Cloudflare Pages** (project: `nod-pages`) and lives at **https://normalizationofdeviance.band**.

There is no auto-deploy from GitHub. You deploy manually when you want changes to go live.

## TL;DR

From the project root (`/Users/richwasserman/projects/nerdcore-album`):

```bash
node build.js && npx wrangler pages deploy dist --project-name=nod-pages --branch=main
```

That's it. ~10 seconds. Site updates instantly worldwide.

## Step-by-step

1. **Make your change.** Edit lyrics under `src/lyrics/`, copy in `src/tracks.json`, design in `src/styles.css`, etc.

2. **Build the static site.** This compiles `src/` + `content/` into `dist/`:
   ```bash
   node build.js
   ```
   Expected output: `✓ Built 7 track pages + index → dist/` and `Audio: 7/7  Art: 7/7`.

3. **Deploy `dist/` to Cloudflare Pages:**
   ```bash
   npx wrangler pages deploy dist --project-name=nod-pages --branch=main
   ```
   Expected output ends with: `✨ Deployment complete! Take a peek over at https://<hash>.nod-pages.pages.dev`. The production URL (`https://normalizationofdeviance.band` and `https://nod-pages.pages.dev`) updates within seconds.

4. **Commit the source change to GitHub** (so the source of truth matches what's deployed):
   ```bash
   git add -A && git commit -m "describe your change" && git push
   ```
   Note: pushing to GitHub does NOT redeploy. The deploy in step 3 is what makes it live.

### About the "commit-dirty" warning

If you have uncommitted changes when you deploy, wrangler will print:
```
To silence this warning, pass in --commit-dirty=true
```
**This is harmless** — the deploy still happens and works fine. Wrangler just notices that your working tree differs from the last git commit. You can ignore it. (If you want to silence it, add `--commit-dirty=true` to the deploy command; or just commit your changes before deploying.)

## Verifying after deploy

Quick sanity check from any terminal:

```bash
curl -sI https://normalizationofdeviance.band/ | grep -E "^(HTTP|content-type)"
```

Should return `HTTP/2 200` and `content-type: text/html`.

## If wrangler isn't authenticated anymore

Cloudflare OAuth tokens can expire. If `npx wrangler pages deploy` complains about auth:

```bash
npx wrangler login
```

Browser opens, click **Allow**, done. Run the deploy command again.

To check current auth status:
```bash
npx wrangler whoami
```

## Analytics dashboard (local only, never deployed)

Pull traffic stats from Cloudflare and generate a local dashboard:

```bash
python3 tools/analytics.py --days 14 --html --open
```

- `--days N` — window size (default 14). Adaptive groups are capped at 1-day per query, so the script iterates one day at a time and aggregates client-side.
- `--html` — write `analytics-output/index.html` (gitignored, never deployed).
- `--open` — open the generated dashboard in your browser.

Without `--html`, prints a console-only report.

**Auth:** the script reads `~/.cloudflare-analytics-token` (chmod 600, *outside* the repo). To get a token: see https://dash.cloudflare.com/profile/api-tokens — needs `Account.Account Analytics:Read`, `Zone.Analytics:Read`, `Zone.Zone:Read`. Token is read-only and revocable.

**Output is gitignored** — `analytics-output/` and `.cloudflare-analytics-token` are both in `.gitignore`. The HTML embeds traffic numbers but **no secrets**.

## Common edits

| You want to | Edit | Then |
|---|---|---|
| Fix a typo in lyrics | `src/lyrics/<slug>.txt` | rebuild + deploy |
| Change a one-liner / sin description | `src/tracks.json` | rebuild + deploy |
| Replace an audio file | `content/<original-name>.mp3` (keep filename) | rebuild + deploy |
| Replace cover art | `content/<slug>_1800x1800.png` then run the `sips` command in this file's history to recompress | rebuild + deploy |
| Adjust design | `src/styles.css` | rebuild + deploy |
| Update the disclosure / contact / blog link | `src/tracks.json` (`album` block) | rebuild + deploy |

## Don't deploy if

- The build prints `[warn] no audio for <slug>` or `[warn] no art for <slug>` — fix the missing file first.
- Tests broken (n/a here, no test suite).
- You're committing WAV files. They're gitignored on purpose; the site only serves MP3.
